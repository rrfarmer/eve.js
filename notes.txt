whenever the SERVER logs [LOG]: [CallReq] alert::BeanCount(), it means the client has crashed and is sending an error report
if client gives error: Formatted exception info: TypeError: 'NoneType' object is not iterable, that means that service is not implemented yet
return {
  type: "object",
  name: "util.KeyVal",
  args: {
    type: "dict",
    entries: [["Index", { type: "token", value: "util.Row" }]],
  },
};
the above returns an object that has an Index property, which is a token "util.Row".
this solves the issue where the client says ... has no attribute 'Index'
